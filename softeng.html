Software Application for Brigham & Women's Hospital

Project Presentation

Project Description

For my software engineering class, in a team of 10 people we applied Agile Scrum methodology for 7 weeks. In this time, we developed a prototype desktop application for the hospital employees of Brigham & Women's Hospital to access and use. The application focuses on creating and managing Service Requests such as food, gift, and medical deliveries, as well as laundry and transportation services. Additionally, a database of locations, medical equipment, medical equipment service requests, and employees is maintained and implemented in an easy-to-use graphical interface.

As part of the Agile methodologies we applied, we conducted daily scrums, sprint plannings and sprint reflections. We also had roles, including front-end engineers, back-end engineers, project manager, product owner, and others. I served as Assistant Lead Software Engineer, coordinating other software engineers, and back-end systems lead. I also served as an algorithms specialist.

Contributions

Briefly, I worked on the following:

  • Applied Agile development methodologies and software design patterns in Java in team of 10 people.
  • Coordinated software engineers as Assistant Lead Software Engineer and performed code reviews.
  • Implemented embedded, client-server and cloud databases in MySQL and MongoDB using JDBC driver.

In more details, I created low-level CSVReader and CSVWriter to read from and write to CSV files to and from Object collections. Designed and implemented Parser classes to convert between List of Strings to Collection of Objects. Created initial design of Database Manager and implemented. Redesigned, refactored and implemented data-access object (DAO) pattern for Embedded Database Management System using Apache Derby SQL. Implemented backup directory system. Worked on JAR compatibility between Windows and Mac. Expanded DAO design into using Facade pattern for Embedded Database. Implemented more DAOs for new Service Requests and Tables. Designed and implemented Facade pattern for Restore/Backup system. Implemented Client-Server Database and feature to switch between Database connections. Designed and implemented Cloud-Based NoSQL (MongoDB) Database using DAO, Facade and Delegation patterns. Worked on setting up the cloud-based database in MongoDB Atlas. Worked on switching between all three databases. Worked on improving remote database performance. Assisted on setting up a second camera for the hardware special feature. Worked on getting cameras working across OS (Mac and Windows).
